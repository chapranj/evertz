OVERVIEW

XPS is Evertz's streaming platform that provides braodcast-quality video encoding at ultra-low latencies for mission-critical applications, supporting up to 4x simultaneous 1080p50/60 HD video encodes or decodes. It can be configured in software to switch between either a four-channel encoder, four-channel-decoder or a combination of two encoders and two decoders. Aditionally, the XPS Series supports up to 2x 2160p50/60 4K UHD video resolutions (requires 4K APP to be installed) with 16x channels of audio, with support for different compression standards such as HEVC or H.264 over any IP network.

Features & Benefits

  • Real-time ultra-low latency with highest video quality for 24/7 up-time applications.
  • High density with quad-channel encode or decode, standalone or rack mount chasis options.
  • Native support of protocols such as SRT, ZIXI, RIST, HLS, RTMP/RTMPS for streamng over lossy networks like the Internet.
  • High availability with dual power supply options.
  • Multiple form-factors to suit all applications.

Applications

  • 4K remote contribution
  • Monitoring return feeds
  • Remote operation
  • Remote monitoring
  • Esports
  • Live interviews
  • Mission command and control
  • IPTV distribution

The XPS EDGE can be ordered with various package options to Encode and/or Decode using the SDI Ports.

  • 2 Decoders (2D)

CONTROL PORT IP CONFIGURATION

The XPS unit should be labeled with default control IPs. Use control port 2 for mangement. The card IP can be configured either through the USB port or through ADV UI.

  1. Remove the card from the box.
  2. Record the IP on the label.
  3. Connect Control port 1 of the card to the Workstation.
  4. Connect Control port 2 of the card to the network or router.
  5. Use Option 1 to configure the card IP, use Option 2 if Option 1 fails.
  6. Option 1 pre-requisites: Workstation with RJ45 Ethernet network interface or external Ethernet adapter.

Option 1: Configure IP Through Web GUI

  1. Click on the Settings button.
  2. Click on the Network tab.
  3. Configure IP settings for control port 2:
    1. Enter desired IP settings.
    2. Alternatively, if DHCP server has been set on your network, you may enable DHCP for automatic port selection.
  4. Click the Apply button to finialize your changes.

Option 2: Configure IP Through USB Serial Port

  1. Connect a USB Type-C into the port in the card's front edge, connect the other end to your workstation. Workstation without a USB port will require an external USB adapter.
  2. Open a terminal software (like Tera Term) and configure the serial port settings as below.

    3. Note: the COM port might be different.

    • Mac users might not have access to TeraTerm. Please follow these steps:
      • Once the serial cable is connected to a Mac workstation, open up a terminal.
      • Type in the command: ls /dev/tty.*
      • XPS should be listed as tty.usbserial-XXXXXX
      • Type in the next command to connect to the serial port: screen /dev/tty.usbserial-XXXXXX 115200
  3. Make sure you are connected to the right COM/tty.
    • The terminal will display "Welcome to XPS-EDGE-M Platform". Network configurations are only accessible for "-M Platform" serial port.
  4. Login to serial term with username/password: customer/customer
  5. Navigate to Network setup and configure Control port 2.
  6. Press X to save your changes and exit.

ENCODER CONFIGURATION

CONFIGURE ENCODER CONTROL

  1. Choose which Encoder Channel to configure.
  2. Click on the Control tab.
  3. Configure Video Encoder.
    1. Choose Video Codec.
    2. Enable/Disable Deinterlacer.
      1. Deinterlacer will always be ON for H264 encode if the SDI Input is HD Interlaced.
      2. Deinterlacer is configurable for HEVC encode of the SDI Input is HD Interlaced.
    3. Toggle Intra Refresh On/Off.
      1. Off: Manually control GOP Structure & GOP Size.
      2. On: Optimized for minimum latency; P frames only, no manual selection for GOP Structure & GOP Size.
  4. Configure Audio.
  5. Configure VANC.
  6. Assign PID numbers, Program Number and TS ID.
  7. Click the Apply button to finialize your changes.

Note: Please make sure the right Encoder Mode is selected.

CONFIGURE ENCODER OUTPUT

  1. Choose the Encoder Channel.
  2. Click on the Output tab.
  3. Configure Output Streaming Parameters.
    1. Choose Output Shaping.
      1. CBR: Constant Bitrate
      2. CVBR: Capped Variable Bitrate
      3. VBR: Variable Bitrate
    2. Enter Bandwidth Overhead.
  4. Choose the TS Encapsulation Type.
  5. Enter the TTL.
  6. Enable/Disable UDP Output through SFP1 and/or SFP2, enter multicast IP address and UDP Port Number.
  7. Click the Apply button to finialize your changes.

RTMP Output Protocol

  1. Choose the Encoder Channel.
  2. Click on the Output tab.
  3. Select RTMP as the Output Protocol in the Control Output Protocol drop-down menu.
  4. Enable Main or/and Redundant Stream.
  5. Set RTMP URL.
  6. Set Stream Name, username, passphrase, and flash version.
  7. Click the Apply button to finialize your changes.

Note: RTMP will not work with interlaced input and/or HEVC codec. Enabling RTMP will automatically switch codec to H264 and audio to AAC2.0.

SRT Output Protocol

  1. Choose the Encoder Channel.
  2. Click on the Output tab.
  3. Select SRT as the Output Protocol in the Control Output Protocol drop-down menu.
  4. Enable Main or/and Redundant Stream.
  5. Choose the Network Interface for SRT - Control port 1 OR Control port 2.
  6. Choose SRT Connection Mode - Caller, Listener, and Rendezvous.
    1. For Caller mode, enter Address, Destination Port, and toggle Source Port Mode (if enabled, enter Source Port number).
    2. For Listener mode, enter Port number.
    3. For Rendezvous mode, enter Address, Port number, and Source Port number.
  7. Enter Output TTL, SRT Latency, and enable/disable Encryption.
  8. Click the Apply button to finialize your changes.

Note: The minimum recommended latency can be monitored in the SRT Main Section in the image below.

RIST Output Protocol

  1. Choose the Encoder Channel.
  2. Click on the Output tab.
  3. Select RIST as the Output Protocol in the Control Output Protocol drop-down menu.
  4. Enable Main or/and Redundant Stream.
  5. Choose the Network Interace for RIST - Control port 1 OR Control port 2.
  6. Choose RIST Transmitter Connection Profile - Simple or Main.
  7. Configure RIST Connection Parameters:
    1. For Simple Profile, Destination Port number, Stream ID, Latency, Session Timeout, enable/disable DTLS, DTLS Peer Verify, pre-Shared Key Type, and Null Packet Deletion.
    2. For Main Profile:
      1. Choose RIST Connection Mode - Server or Client.
      2. For Server mode, enter Destination Port number, Stream ID, Latency, Session Timeout, enable/disable DTLS, DTLS Peer Verify, pre-Shared Key Type, and Null Packet Deletion.
      3. For Client mode, enter Destination IP, Destination Port number, Stream ID, Latency, Session Timeout, enable/disable DTLS, DTLS Peer Verify, pre-Shared Key Type, and Null Packet Deletion.
  8. Click the Apply button to finialize your changes.

RIST Encoding Status can be monitored in the Monitor page on the Streaming Output Monitor sections as seen below.

ZIXI Output Protocol

  1. Choose the Encoder Channel.
  2. Click on the Output tab.
  3. Select ZIXI as the Output Protocol in the Control Output Protocol drop-down menu.
  4. Enable Main or/and Redundant Stream.
  5. Choose the Network Interface for ZIXI - Control port 1, Control port 2, or alternatively, the user can enable Bond Network Interfaces.
  6. Enter ZIXI Connection details. Enter Broadcast host details (Host, Alternate Host, Port Number, and Stream Credentials).
  7. Enter ZIXI Feeder Settings. Select Max Latency, Bitrate Adaptive Mode, FEC Overhead, DTLS Certificate Error Ignore, and Encryption Type.
  8. Click the Apply button to finialize your changes.

ZIXI Streaming Status can be monitored in the Monitor page on the Streaming Output Monitor sections as seen below.

HLS Output Protocol

  1. Choose the Encoder Channel.
  2. Click on the Output tab.
  3. Select HLS as the Output Protocol in the Control Output Protocol drop-down menu.
  4. Enable Main or/and Redundant Stream.
  5. Enter HLS Stream Playlist URL address.
  6. Enter Username, Password, and Segment Duration.
  7. Click the Apply button to finialize your changes.

CONFIGURE DECODER INPUT

SRT Input Protocol

  1. Choose which Decoder Channel to configure.
  2. Click on the Input tab.
  3. Choose SRT as the Input Protocol.
  4. Choose a Failover Mode. If Manual is chosen, select Main, Backup or Manual.
  5. Choose Input Interface - Control port 1 or Control port 2.
  6. Choose SRT Connection Mode - Caller, Listener, and Rendezvous.
    1. For Caller mode, enter Address and Port number, and toggle Source Port Mode (if enabled, enter Source Port number).
    2. For Listener mode, enter Port number.
    3. For Rendezvous mode, enter Address, Destination Port number, and Source Port number.
  7. Enable/Disable Encryption and Enter SRT Latency.
  8. Click the Apply button to finialize your changes.

RIST Input Protocol

  1. Choose which Decoder Channel to configure.
  2. Click on the Input tab.
  3. Choose RIST as the Input Protocol.
  4. Choose a Failover Mode. If Manual is chosen, select Main, Backup or Manual.
  5. Choose Input Interface - Control port 1 or Control port 2.
  6. Choose RIST Receiver Connection Profile - Simple or Main.
  7. Configure RIST Connection Parameters:
    1. For Simple Profile, Destination Port number, Stream ID, Latency, Session Timeout, enable/disable DTLS, DTLS Peer Verify, pre-Shared Key Type, and Null Packet Deletion.
    2. For Main Profile:
      1. Choose RIST Connection Mode - Server or Client.
      2. For Server mode, enter Destination Port number, Stream ID, Latency, Session Timeout, enable/disable DTLS, DTLS Peer Verify, pre-Shared Key Type, and Null Packet Deletion.
      3. For Client mode, enter Destination IP, Destination Port number, Stream ID, Latency, Session Timeout, enable/disable DTLS, DTLS Peer Verify, pre-Shared Key Type, and Null Packet Deletion.
  8. Click the Apply button to finialize your changes.

ZIXI Input Protocol

  1. Choose which Decoder Channel to configure.
  2. Click on the Input tab.
  3. Choose ZIXI as the Input Protocol.
  4. Choose a Failover Mode. If Manual is chosen, select Main, Backup or Manual.
  5. Choose Input Interface - Control port 1 or Control port 2.
  6. Enter ZIXI Connection details. Enter Broadcast host details (Host, Alternate Host, Port Number, and Stream Credentials).
  7. Enter ZIXI Feeder Settings. Select Max Latency, Bitrate Adaptive Mode, FEC Overhead, DTLS Certificate Error Ignore, and Encryption Type.
  8. Click the Apply button to finialize your changes.

PRESETS

Presets function in Encoder and Decoder Channels to allow a user to Save, Load, Import and Export Channel configurations. Presets can be used for backups and to quickly switch configurations of a channel. To see available Preset options, click the Preset drop-down menu.

  1. Click the Load button to load a saved/imported preset to the channel.
  2. Click the Save button to save a channel preset.
  3. Click the Delete button to delete a saved channel preset.
  4. Click the Import button to import a channel preset from your workstation to XPS.
  5. Click the Export button to export a channel preset from XPS to your workstation.

IMPORT & EXPORT SYSTEM CONFIGURATION FILES

Users can Import & Export configuration files to save a copy of the current XPS configuration to be used for backup or to copy the configuration to different units. Through Advanced UI, users can use the Import & Export Configure File found on the Systems Management tab in the Settings menu as shown below.

Export a configuration file:

  1. Click on the System Management tab in the Settings menu.
  2. Click on the Download button and choose the desired folder location to save the JSON file.

Import a configuration file:

  1. Click on the System Management tab in the Settings menu.
  2. Click on the Choose File button and navigate to the folder where the configuration file has been saved, click Open to select the saved JSON file.
  3. Click on the Upload button to sync with the saved Configuration File.
  4. Click the Apply button to finialize your changes.

FIRMWARE UPGRADE

Using the Advanced UI® on a web browser is the fastest and recommended procedure to laod the firmware onto the XPS Series products.

FIRMWARE UPGRADE USING ADV UI®

On the top of the web page for the XPS-EDGE-HW interface, there is an Upgrade button. The Upgrade tab is used to check the current firmware version and upload the latest firmware.

Note: Contact Evertz to get the latest firmware file.

Selecting the Upgrade tab will open a new window, as shown below, where the current firmware version is shown. Should the firmware version be outdated, the user will need to downlaod the firmware image file.

Click Choose File and browse to locate the image file.

Once selected:

  1. Navigate to the right folder and click Open to select the new firmware file.
  2. Click Upgrade and watch the progress bar for the status. Once compelted, the device will automatically restart.

Note: An alternate method to upgrade firmware is mentioned in the next section.

STEPS TO SAVE MULTIPLE FIRMWARES

  1. Click on the Settings menu.
  2. Click on the System Image tab.
  3. Click on Choose File and click Open.
  4. Click on the Upload button and select Yes on the Confirmation pop-up.
  5. Repeat the same steps to add multiple firmware.
  6. Choose the firmware you want and click on Upgrade as shown below.
